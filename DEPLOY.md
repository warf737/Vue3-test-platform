# Инструкция по деплою на GitHub Pages

## Быстрый старт

1. **Создайте репозиторий на GitHub** (если еще не создан)

2. **Включите GitHub Pages:**
   - Перейдите в `Settings` → `Pages`
   - В разделе `Source` выберите `GitHub Actions`

3. **Проверьте настройки workflow:**
   - Откройте `.github/workflows/deploy.yml`
   - Убедитесь, что имя репозитория указано правильно
   - По умолчанию используется `${{ github.event.repository.name }}`, что автоматически подставит имя вашего репозитория

4. **Запушьте код в ветку `main` или `master`:**
   ```bash
   git add .
   git commit -m "Initial commit"
   git push origin main
   ```

5. **Проверьте деплой:**
   - Перейдите в `Actions` в вашем репозитории
   - Дождитесь завершения workflow `Deploy to GitHub Pages`
   - После успешного деплоя приложение будет доступно по адресу:
     `https://ваш-username.github.io/имя-репозитория/`

## Настройка для кастомного домена

Если вы используете кастомный домен, установите `BASE_URL: '/'` в workflow файле.

## Ручной запуск деплоя

Вы можете запустить деплой вручную:
1. Перейдите в `Actions` → `Deploy to GitHub Pages`
2. Нажмите `Run workflow`
3. Выберите ветку и нажмите `Run workflow`

## Устранение проблем

### Проблема: 404 при переходе по прямым ссылкам

**Решение:** Убедитесь, что в `vite.config.ts` правильно настроен `base` URL, соответствующий имени вашего репозитория.

### Проблема: Стили не загружаются

**Решение:** Проверьте, что `BASE_URL` в workflow совпадает с именем репозитория.

### Проблема: Workflow не запускается

**Решение:**
- Убедитесь, что GitHub Pages включен в настройках репозитория
- Проверьте, что файл `.github/workflows/deploy.yml` находится в репозитории
- Убедитесь, что вы пушите в ветку `main` или `master`
